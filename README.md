   网上看到宇航员飞船回地球的网上文章时，找个太平洋的尼莫点来降落，于是百度查找，看了【难抵极/尼莫点】的相关知识，于是要，想找下有没有相关软件来看下，结果一时找不到一星半点，于是，自已开始冥想构思，作为小软件来用下，由于是小软件，只能是【近似计算】的。
   【由于没有任何别的软件来参考，一切全凭我的想像来开发的】。

   先载入一张南极地图（网上下载的，别的地图自已另行下载）
地图的最大尺寸是长与宽都是3000的像素：3000*3000，以像素为单位的。
坐标与【距离】，全是以像素为单位的。坐标是屏幕坐标，原点在左上角，X轴向右，Y轴向下(不是向上)
   点击描绘海岸线，描绘南极地图，这时，发现，海岸线是以实际大陆还是以永久冰川还是以季节性的冰川，搞不清，是尝试玩下的，不作太多要求，描绘下来，就设离岸距离为L=30试下，太小，再试成100，再试，仍太小，于是试到170，175，180，于是接近南极难抵极。
   发现，与网上下载的标明的有误差，也不知是不是平面还是海岸线的原因，实际我的小软件求出来的结果是，“难抵极”的抵字的扌的顶上的最顶点的位置。
   于是，点击这个点的坐标，刷新下，以这个点为圆心来画圆，设圆半径为175试看下效果如何。
   别的地图一样这样的玩法

【算法说明】：
就是以点为圆心来画圆
先描绘海岸线，然后以海岸线上的每点来画圆，就是这样的最简单算法
验证结果也是这样，以难抵极为圆心来画圆来验证结果的。
